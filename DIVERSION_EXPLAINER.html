Diversion Route Planner

A cascade-aware routing system that tells Bengaluru police officers exactly which alternate road to use — and which ones to avoid — when a corridor is blocked.

The Problem

What's broken today?

Every day in Bengaluru, roads get blocked — accidents, VIP movements, cricket matches, political rallies. When that happens, police manually decide which alternate road to send traffic on.

The issue: They pick an alternate road based on experience. But that alternate road might already be congested. Sending thousands of diverted cars onto a busy road creates a second jam — this is called a cascade failure.

No system exists today that says: "Don't divert to Magadi Road right now — it already has a cluster of incidents. Use ORR West instead."

The Solution

What we built

💡 Simple Analogy

Think of it like Google Maps for police — but instead of routing individual cars, it routes entire traffic streams. And instead of just finding the shortest path, it also checks: "Is the alternate road already jammed?" If yes, it finds the next best option.

We built a system with three parts that work together:

1
Road Network

Map Bengaluru's roads as a graph

We took all the major corridors from the police dataset (Mysore Road, Bellary Road, ORR, Tumkur Road, etc.) and turned them into a network of connected dots and lines — junctions are the dots, roads are the lines. Roads with more historical accidents are given a higher "cost" so the system naturally avoids them when choosing a diversion.

2
Hotspot Detection

Find where congestion is happening right now

We run a clustering algorithm called DBSCAN on all 8,139 events in the dataset. It automatically groups nearby incidents into "hotspot zones" — like a heat map that shows which areas of the city are currently busy. We found 29 such clusters across Bengaluru.

3
Cascade-Aware Routing

Find the best alternate route — safely

When a road is blocked, we find the top alternate routes using an algorithm called Dijkstra's (the same one Google Maps uses). Then we apply the cascade filter: any route that passes through a current hotspot zone gets rejected. The first clean route becomes the primary diversion, the second becomes the backup.

Deeper Dive

How the routing actually works

Here's a simple picture of what the road network looks like. Each circle is a major junction. The lines are roads. The numbers represent how "busy" each road is historically.

In the diagram above — when Mysore Road is blocked (dashed red), the system finds that the direct path goes right through a congestion hotspot. So instead it picks Magadi Road (solid green) which is parallel, less busy, and hotspot-free.

The cascade awareness step — explained simply

The classic mistake: A crash on Mysore Road → police redirect all traffic to Magadi Road → Magadi Road also has a cluster of 50 incidents today → now Magadi Road is jammed too. You've created two jams instead of one.
What our system does: Before recommending Magadi Road, it checks: "Is there a current hotspot cluster within 1.2 km of any junction on this route?" If yes → reject it, try the next best path. It keeps checking until it finds a route that's genuinely clear.

Visual Output

What the interactive map shows

Each scenario generates an .html map file you can open in any browser. Here's what the colours mean:

Red dashed line — The blocked corridor. Officers should barricade all entry points.
Solid green line — Primary diversion. This is the recommended alternate route.
Amber dashed line — Secondary (backup) diversion if primary gets congested.
Purple circles — Live congestion hotspots (detected by DBSCAN). Any route passing through these was automatically rejected.
Gray lines — All other background corridors for reference.

The maps are saved as diversion_map_s1.html, diversion_map_s2.html, diversion_map_s3.html — open any of them in Chrome/Safari to explore interactively.

Differentiator

Why this is different from just "find shortest path"

Any GPS can find the shortest path. The differentiator here is the cascade awareness layer — the ability to reject routes that will overflow onto already-congested roads, which no existing Bengaluru police tool does today.

On top of that, the edge weights are built from real historical event data — so the system already "knows" that Mysore Road (743 events) is riskier than Magadi Road (245 events) and prices them accordingly, even before any filtering.

The flywheel: As more events are logged into ASTRAM, the hotspot detection gets sharper, the edge weights get more accurate, and the routing recommendations get smarter — automatically.
